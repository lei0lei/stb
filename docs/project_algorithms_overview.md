# 核心算法总览

这个项目的算法分散在多个独立库中。它们共同特点是：实现尽量自包含，依赖少，优先使用 C 语言和宏配置来适配不同使用场景。

## 图像读取：`stb_image.h`

`stb_image.h` 的主线逻辑是先识别输入格式，再进入对应解码器。它支持 JPEG、PNG、TGA、BMP、PSD、GIF、HDR、PIC、PNM 等格式。每个格式解码器负责解析文件头、校验尺寸和通道信息、解压或反变换像素数据，最后统一转换成调用者要求的通道数。

JPEG 部分涉及 Huffman 熵解码、DCT 系数反量化、IDCT、YCbCr 到 RGB 转换和采样重建。PNG 部分涉及 chunk 解析、zlib/deflate 解压、scanline filter 反滤波、调色板和透明度处理。GIF 依赖 LZW 解码和帧合成逻辑。整体算法重点是把多种文件格式差异收敛到统一的像素缓冲区输出。

## 图像写出：`stb_image_write.h`

写出库把内存中的像素数据编码成 PNG、BMP、TGA、HDR、JPG 等格式。BMP/TGA 相对直接，主要是文件头和像素排列转换；PNG 需要对扫描线做过滤并压缩；JPG 写出包含颜色转换、分块 DCT、量化和熵编码。它的设计目标是方便生成文件，而不是覆盖每个格式的全部高级特性。

## 图像缩放：`stb_image_resize2.h`

图像缩放的核心是重采样。算法会根据目标尺寸和源尺寸计算每个输出像素对应的源像素范围，并使用滤波器权重累积结果。放大和缩小时的滤波半径不同，缩小时需要覆盖更大源区域以减少混叠。实现中还需要处理边界、通道布局、色彩空间、alpha 预乘以及不同像素类型。

## 字体解析和光栅化：`stb_truetype.h`

TrueType 字体算法分为解析和渲染两层。解析层读取字体表目录、字形索引、轮廓点、曲线控制点、字距和度量信息。渲染层把二次 Bezier 曲线轮廓转成边列表或采样结果，再通过扫描线/覆盖率计算得到灰度位图。打包字体贴图时还会结合矩形装箱，把多个字形放进一张 atlas。

## 音频解码：`stb_vorbis.c`

Vorbis 解码包含 Ogg 页解析、码本读取、Huffman/码本解码、floor/residue 重建、inverse MDCT 和窗口重叠相加。文件中的状态机负责从比特流逐步恢复频域数据，再转换成 PCM 采样。算法复杂度主要来自 Vorbis 规范本身：它既要维护包级别解析状态，也要处理声道耦合、块大小切换和浮点/整数输出。

## 数据结构：`stb_ds.h`

`stb_ds.h` 提供动态数组和哈希表。动态数组通过在用户指针前方隐藏一段 header 保存长度和容量，宏操作看起来像普通数组，实际会在容量不足时重新分配。哈希表使用开放寻址策略保存键值对，并通过宏生成类型友好的访问形式。核心取舍是用宏换取 C 语言里接近泛型容器的使用体验。

## 矩形装箱：`stb_rect_pack.h`

矩形装箱用于把多个小矩形放入一个大矩形中，常用于纹理 atlas。实现采用 skyline 思路：维护当前已占用区域的轮廓线，放置新矩形时寻找合适位置并更新轮廓。它不是全局最优算法，但速度快、实现小、效果足够适合实时工具和资源构建流程。

## 噪声：`stb_perlin.h`

Perlin/simplex 噪声通过格点伪随机梯度、插值和平滑函数生成连续噪声值。调用者输入坐标和可选 wrap 参数，算法返回稳定的伪随机连续值。多频率叠加后可生成地形、纹理、云雾等自然感图案。

## 文本编辑状态机：`stb_textedit.h`

文本编辑库不负责绘制 UI，而是维护光标、选区、插入、删除、撤销重做和键盘操作。调用者提供访问字符、布局和修改文本的回调，库内部根据输入事件更新编辑状态。这种设计让同一套编辑逻辑可嵌入游戏 UI、工具界面或自定义文本框。

## 体素和瓦片：`stb_voxel_render.h`、`stb_tilemap_editor.h`、`stb_herringbone_wang_tile.h`

体素渲染库把块状世界数据转换成可渲染的网格，并通过面剔除、几何合并或材质信息减少绘制成本。瓦片编辑器围绕地图数据、选择状态、绘制操作和 UI 事件组织。Herringbone Wang tile 生成器使用带边约束的瓦片组合规则，让局部 tile 拼接形成更自然的大地图。

## 词法、格式化和辅助算法

`stb_c_lexer.h` 把输入文本扫描成标识符、数字、字符串、运算符等 token，供简单语言或配置解析器使用。`stb_sprintf.h` 实现高性能格式化输出，需要解析格式串、处理整数/浮点转换和缓冲写入。`stb_divide.h` 关注更符合数学直觉的整数除法和取模语义。`stb_connected_components.h` 维护网格连通性，用于在格子变化时增量判断可达区域。
